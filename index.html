Application Overview

The concept of this project is a web application designed for users to track their daily diet and intake. Users can enter their meals and specify the portion sizes for each food item. The app provides detailed feedback on caloric intake, protein, vitamins, and other crucial nutrition data. It allows users to review their nutritional data on a daily, weekly, or even monthly basis for an overlook of their diet habits.

Technology Overview

  • Languages: Go, HTML, CSS, JavaScript
  • Libraries: GraphQL with Go
  • Platform: Docker
  • Hosting: Static Web Server (hosted on Azure) with basic .html, .js, and .css files

Feature List

  1. Food intake logging for each meal with portion size details
  2. Daily caloric intake tracking
  3. Nutrition breakdown (calories, sugars, etc.)
  4. Weekly and monthly summary views
  5. Simple GraphQL API for handling food data

Future Features

  • Ability to set personal dietary goals
  • Meal planning recommendations based on past data
  • Enhanced data security with user authentication

Basic Technological Features

  • Basic web server hosting with static HTML, CSS, and JavaScript files
  • Integration of a simple public API (FoodData Central)
  • Setup of Go Docker container for the application

Advanced Technological Features

  • Custom GraphQL API with Go
  • Implementation of a simple, insecure API
  • Weekly and monthly aggregated data reporting
